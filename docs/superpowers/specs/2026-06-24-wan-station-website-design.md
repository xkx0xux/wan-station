# わんステーション 公式サイト 設計書（v1）

作成日：2026-06-24　／　対象：わんステーション 公式サイト 初版（ランディング＋将来ページの器）

---

## 1. これは何か（目的）

NPO法人わんステーション・ジャパンが運営する「滋賀発・保護犬支援プラットフォーム＝わんステーション」の**公式サイト**を作る。

- 役割：**わんステの世界観と活動を伝える公式サイト**。今は **Readyfor のクラウドファンディング（CF）応援への誘導を主役**にする。
- 将来：寄付・会員・活動報告を足していける**土台**にする（最初に「公式サイト＋CF応援」を選択済み）。

### 成功の基準
- 初めて見た人が「何の団体か」「何をしているか」「なぜ応援すべきか」を1分で理解できる。
- 「クラウドファンディングで応援する」ボタンが各所から自然に押せる。
- **奇をてらわず、きちんとした団体だと一目で伝わる（安心・信頼）**。AIっぽい派手さは出さない。

## 2. 決定事項（ブレストで合意済み）

| 項目 | 決定 |
|---|---|
| サイトの形 | **C：ハイブリッド** … 縦長トップ1枚＋将来ページの器 |
| 見た目の方向 | **安心・信頼**。紺基調×ゴールドの細い効かせ×明朝の見出し。派手な装飾なし |
| 写真・素材 | 本物写真はまだ無い。**実写系のフリー素材**（AI感の出ない犬写真）＋設計図トーンのイラスト。ロゴは資料の金色の犬をベースに |
| 公開 | まずは**手元（ローカル）で確認**。OK後に公開は別途（将来 Render 無料を想定） |
| お問い合わせ | **メール＋SNSリンクのみ**（フォーム無し・無料・壊れない） |
| お金まわり | サイト内に決済・寄付フォーム・会員DB・ログインは**作らない**。寄付/CFは外部 Readyfor に飛ばす＝事故リスクゼロ |

## 5. デザイン仕様

### カラー
| 用途 | 色 |
|---|---|
| ブランド主色（紺） | `#16223C` |
| アクセント（ゴールド） | `#B8862A` 〜 `#C8932E` |
| 背景（オフホワイト） | `#FAF8F4` / 白 `#FFFFFF` |
| 本文テキスト | `#2C3340` |
| 補助テキスト | `#8A8270` |
| 罫線・区切り | `#ECE6DC` |

### タイポグラフィ（Google Fonts・無料）
- 見出し：**Noto Serif JP**（明朝）weight 500–600 … 誠実・伝統・信頼
- 本文：**Noto Sans JP**（ゴシック）… 読みやすさ

### トーン＆作法
- 余白を広く、要素を詰め込みすぎない（上品・端正）
- ゴールドは**細い線・小さな印・下線**で効かせる（金ピカにしない）
- 影は淡く自然に。角丸は中程度。グラデ・グロー・浮遊3Dは使わない（AI感を出さない）
- 写真は**落ち着いた実写犬を大きく1枚**。コラージュ多用しない
- 信頼の根拠（保健所連携／提携病院／お金の流れ全公開）をさりげなく明示

### ロゴ
- 資料（PDF）の**金色の犬マーク**をベースに整えて使用。
- 当面は紺の円＋ゴールドの犬/肉球の簡易マーク（SVG）。後で正式ロゴに差し替え可能な作りにする。

## 6. 技術構成

- **静的サイト**（HTML / CSS / 少量の vanilla JS）。サーバー・DB・ビルド工程なし＝無料・速い・壊れにくい。
- レスポンシブ（PC / タブレット / スマホ）。スマホ最優先で読みやすく。
- JS の役割：スマホメニュー開閉、スクロール追従ヘッダー、スクロールでふわっと表示（無くても全内容が読める＝JSオフでも壊れない）。
- 画像：実写系フリー素材（商用利用OK・帰属不要のもの）。当面はプレースホルダ画像で組み、後で差し替え。
- フォントは Google Fonts から読み込み（無料）。

## 7. コンテンツ（文言の素案）

- ヒーロー見出し（案）：「その命に、続いていく居場所を。」
- ヒーロー補足（案）：「一度きりの善意に頼らない。寄付・買い物・視聴・参加――日常の選択が、保護犬を支え続ける仕組みをつくります。」
- 主要CTAラベル：「クラウドファンディングで応援する」

確定文言は、企画書・FC設計書の内容を要約して私が用意し、マスターが後から微修正できるようにする。

### 後で差し替える情報（v1 は仮の文字＝プレースホルダで作る）
- NPO法人の正式名称・代表者・所在地・設立年月・連絡先メール
- SNS / YouTube の各URL
- **Readyfor の公開URL**（現在はプレビューのため、公開後に差し替え）
- 本物の写真・実績数字（里親成立頭数 等）

## 8. 確認・テスト方法

- ローカルでブラウザ表示確認（PC幅／スマホ幅）。
- リンク切れ・崩れチェック（特に Readyfor・SNS・メール）。
- 文言の事実確認（企画書/FC設計書と矛盾がないか）。
- マスターが実機（Mac／iPhone）で見て最終OK。
